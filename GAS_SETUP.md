# Google Apps Script (GAS) の設定・公開手順

この手順に沿って、今回作成した `Code.gs` をGoogleにデプロイし、アプリと連携させます。

## 1. GASプロジェクトの作成
1. Googleアカウントにログインした状態で、[Google Apps Script (script.new)](https://script.new/) にアクセスします。
2. 「無題のプロジェクト」という新しいプロジェクトが開くので、画面左上のタイトルをクリックして「予約システムAPI」など分かりやすい名前に変更します。

## 2. コードの貼り付け
1. 画面中央のエディタに最初から書かれている `function myFunction() { ... }` を**すべて削除**します。
2. デスクトップの `LINE_mini_app_seminar` フォルダに作成された `Code.gs` ファイルをテキストエディタで開き、中のコードを**すべてコピー**します。
3. GASのエディタに貼り付けます。
4. キーボードの `Ctrl + S` (Macは `Cmd + S`) を押すか、上のフロッピーディスクのアイコン（保存ボタン）を押して保存します。

## 3. Webアプリとしてデプロイ（公開）
1. 画面右上の青いボタン **「デプロイ」** をクリックし、**「新しいデプロイ」** を選択します。
2. 左上の歯車アイコン（種類の選択）をクリックし、**「ウェブアプリ」** にチェックを入れます。
3. 設定項目を以下のようにします：
   - **説明**: 任意（例：v1）
   - **次のユーザーとして実行**: **自分** (`あなたのメアド`)
   - **アクセスできるユーザー**: **全員**
   > ⚠️ 「アクセスできるユーザー」を「全員」にしないと、Webブラウザからのアクセスがブロック・失敗します。
4. 右下の **「デプロイ」** ボタンをクリックします。

## 4. 権限の承認（初回のみ）
1. 「アクセスを承認」という画面が出たらクリックします。
2. 自分のGoogleアカウントを選択します。
3. *「このアプリは Google で確認されていません」* という警告画面が出ます。
   - 左下の **「詳細」** をクリックし、その下に隠れている **「予約システムAPI（安全ではないページ）に移動」** をクリックします。
4. カレンダーへのアクセス許可を求められるので **「許可」** をクリックします。

## 5. WebアプリのURLを取得
1. デプロイが完了すると、「ウェブアプリ」の下に **URL**（`https://script.google.com/macros/s/英数字の羅列/exec`）が表示されます。
2. そのURLの右横にある「コピー」ボタンを押してコピーします。
3. これでGAS側の準備は完了です。

## 6. index.html にURLを設定
1. `index.html` ファイルをエディタで開きます。
2. `<script>` タグの最初の方にある `const GAS_WEB_APP_URL = 'ここにコピーしたURLを貼り付ける';` の部分を探します。
3. コピーしたURLをそこに貼り付けて上書き保存してください。

これで連携が完了し、本物のGoogleカレンダーと連動した予約システムとして動作します！
